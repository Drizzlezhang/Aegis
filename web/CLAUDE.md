# web/CLAUDE.md

## 作用域
- 本文件仅适用于 `web/` 目录。
- 在 `web/` 范围内工作时，需同时遵守仓库根 `CLAUDE.md` 与本文件；若本文件对前端局部事项有更具体约束，则以本文件为准。

## 当前阶段定位
- 当前阶段目标是为前端目录建立第一层局部规则入口。
- 当前阶段只做规则落位，不做物理目录迁移。
- `feature / shared / foundation` 可作为后续结构治理时的职责参考，但当前不视为已承诺的目录形态。

## 前端职责边界
- `web/` 负责前端页面、路由入口、组件装配、前端状态与展示逻辑。
- `web/` 不承载后端 API 编排、交易策略核心、数据抓取与 Agent 主流程职责。
- 涉及后端 contract、模型调用、Agent 编排的改动，应回到仓库根规则与后端边界文档处理，不在 `web/` 内隐式扩散。

## 当前前端局部目录边界
### `app/`
- 路由、页面、layout 与页面级入口。
- 负责 orchestration（页面装配、数据流组织、组件拼接），不沉淀跨页面可复用业务实现。

### `components/`
- 可复用 UI 组件与前端展示组件。
- 共享展示层不应反向依赖路由文件；强业务绑定组件应按域收敛，不应伪装成通用组件。

### `lib/`
- 前端复用逻辑与边界层能力。
- API 请求构造、共享 fetch 行为、主题基础工具等应优先收敛在这里；不反向依赖页面入口。

### `i18n/`
- 国际化文案与语言资源组织。
- 所有用户可见文案应走 i18n，不在组件中硬编码中英文文案。

### `scripts/`
- 若存在该目录，其职责应限定为前端辅助脚本。
- 不作为业务运行时核心。

### `utils/`
- 若存在该目录，其职责应限定为轻量前端工具函数。
- 不引入后端、交易策略或 Agent 侧职责，避免演变为跨层杂项目录。

## 前端局部执行约束
- 前端结构治理优先先收敛逻辑边界，再决定是否做物理迁移。
- 路由层仅保留页面入口职责，不沉淀跨页面通用业务实现。
- API 边界不反向依赖页面文件。
- shared UI / theme / lib 不反向依赖 route files。
- 依赖图应保持无环（DAG）。

## UI 与文案不变量
- 默认支持 `zh-CN` 与 `en`。
- 股票代码、指标缩写、策略术语保持金融语义，不做不自然翻译。
- 行情颜色语义统一遵循“上涨红、下跌绿”，优先复用共享映射，不在组件中各自硬编码。
- 加载态、空态、错误态不可缺失。

## 非目标
- 当前不在 `app/`、`components/`、`lib/`、`i18n/` 下继续新增二级 `CLAUDE.md`。
- 当前不重组 `web/` 目录，不把组件迁入未来候选目录。
- 当前不把文档规则扩展成 UI 改版、页面重构或 API 行为调整任务。
