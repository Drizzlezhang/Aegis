# src/CLAUDE.md

## 作用域
- 本文件仅适用于 `src/` 目录。
- 在 `src/` 范围内工作时，需同时遵守仓库根 `CLAUDE.md` 与本文件；若本文件对后端局部事项有更具体约束，则以本文件为准。

## 当前阶段定位
- 当前阶段目标是为后端目录建立第一层局部规则入口。
- 当前阶段只做规则落位，不做物理目录迁移。
- 当前阶段不处理 `src/` 与顶层 `skills/` 的物理收敛，只先明确 framework 与 implementation 的职责边界。

## 后端职责边界
- `src/` 当前承载后端入口、编排、Skill framework、模型访问及相关后端支撑能力。
- `src/` 不承载前端页面、路由装配、前端展示逻辑与 UI 交互职责。
- 涉及 `web/` 页面编排、主题、i18n 与展示层语义的改动，应回到根规则与前端边界文档处理，不在 `src/` 内隐式扩散。

## 当前后端局部目录边界
### `api/`
- 后端 HTTP 入口、路由装配与协议接入层。
- 负责请求接入、协议适配与路由汇总，不直接承载复杂策略实现，不直接连接供应商模型客户端。

### `agents/`
- 后端编排层与领域执行切片。
- 负责任务流程、阶段协调、多能力调用顺序与领域协作；不承载 HTTP/CLI 协议层细节，不承担 Skill framework 的中心注册治理。

### `skills/`
- Skill framework 层。
- 负责能力接口契约、发现/注册/加载与装配规则；不落地与框架无关的具体业务 Skill 实现。

### `llm/`
- 模型访问层。
- 负责模型路由、客户端调用与供应商封装；不承载业务流程编排，不向入口层暴露绕过编排/能力层的直连路径。

## 当前后端补充边界锚点
### `models/`
- 若当前目录内存在共享后端数据模型，其职责应限定为稳定的数据结构表达。
- 不反向承载入口编排、模型客户端或页面展示职责。

### `backtest/`
- 若存在该目录，其职责应限定为回测相关的后端能力与支撑逻辑。
- 不反向承载 API 入口、Skill framework 中心治理或前端展示职责。

### `config.py`
- 若该文件作为后端配置读取锚点存在，其职责应限定为环境变量驱动的配置装配。
- 不扩散页面逻辑、部署脚本职责或运行时业务编排。

### `cli.py`
- 后端命令行入口。
- 负责参数解析、执行模式选择与结果输出编排，不复制 API/Agent/Skill 内部业务逻辑。

## 后端局部执行约束
- 入口层（`api/`、`cli.py`）只保留接入、适配、触发职责，不直接沉淀复杂策略实现。
- 新增后端流程控制优先收敛在 `agents/` 编排层，不散落在 API/CLI handler。
- 新增 Skill 能力时，先判断是 framework 变更还是 implementation 变更；framework 中心注册逻辑应留在 `src/skills/`，不应散落到 `agents/`、`api/` 等目录。
- 供应商模型客户端必须收敛在 `llm/` 或其受控封装中，入口层不得直接依赖。
- 依赖方向应保持单向，避免循环依赖；入口层不反向成为编排层、Skill framework、模型访问层的被依赖方。
- 当前阶段优先先收敛逻辑边界，再决定是否做物理迁移；不把未来目录形态写成当前承诺。

## 非目标
- 当前不在 `api/`、`agents/`、`skills/`、`llm/` 等二级目录下继续新增 `CLAUDE.md`。
- 当前不重组 `src/` 目录，不做 `src/skills/` 与顶层 `skills/` 的物理迁移。
- 当前不改业务逻辑、API contract、模型路由行为、部署方式或测试链路。
- 当前不把后端边界文档扩展成接口重构、Agent 重写或 Skill 生命周期改造任务。
